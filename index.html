Project Overview


  • My web app, Notorious, aims to compare and constrast NBA statistics with discussion and opinion on Twitter.
  • Twitter hosts an expansive community of NBA fans and experts, and there is non-stop discourse surrounding all the players in the league. Notorious allows users to view a sentiment analysis of the Tweets about a given player at a certain point in time, all while viewing that same player's most updated statistics for refrence.
  • Thus, by enabling users to compare on-the-court performance metrics with an analysis of how positive or negative the Tweets surronding a player are, Notorious digs into the question of how/if Twitter and the proliferation of social media affects NBA atheletes.
  • Do the players with the best stats have the best sentiment score and vice versa, are there certain exceptions to the rule, or is NBA Twitter so brutal in their opinions that the on-court performance of a player holds little influence? Head over to the website to find out for yourself!

Methods


  • The web app is designed to have a responsive frontend (Vue.js and hosted on Gitlab) that interacts with a backend server (Heroku) as well as a Realtime Database (Firebase) that hosts information about all of the users. Using the database not only allows for persistence of data, but also sharing of data.
  • Users can log in via Google OAuth, which then enables them to interact with the database and experiment with the APIs working in the backend.
  • On the Create page, users can search for all NBA players and create player cards for players of their choice. The player cards contain the most updated statistical averages from that player's career.
  • Notorious uses rapidapi's API-NBA to search for NBA players and retreive their stats. Users can also choose to display a sentiment analysis from the Tweets about that player on a given player card. Notorious uses the Twitter API up to 100 of the most recent tweets about a player, runs each Tweet through IBM Watson's Natural Language Understanding API , and then displays that score on the player card. The score ranges from -1 to 1, with -1 being the most negative and 1 being the most positve.
  • Users can refresh the information on a player card at any time, save cards to their favorites, and publish cards to the explore page for all other users to see. Users can also change their display name which is attributed to the cards they publish.
  • Non logged in users can only view the explore page, while admin users have all the priveleges of logged-in users along with the ability to see all login records from their profile page.
  • When changes are made on the frotend, a frontend data store calls the backend server which in turns calls the respective external APIs if neccesary and then sends data to the backend data store so the database is updated. The changes to the database are then returned to the frontend data store which the Vue app responsively reads from.
